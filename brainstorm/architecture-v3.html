Data Agent Pool — general-purpose, stateless, horizontally scalable
Each data agent is identical — it receives a validated, redacted request from the repo agent and executes it.
Pillar-awareness lives in YAML config, not in data agents. Multiple agents handle concurrent specialist requests in parallel.

Single control plane: YAML. Each pillar YAML declares which specialists to call, their analytical focus, table scope, prompt overlays, and the list of blocked data fields (compliance). The orchestrator reads compliance constraints before routing — no specialist ever receives a sub-question that would require blocked data. Adding a new pillar or changing a specialist's emphasis = one YAML edit, no Python changes.
