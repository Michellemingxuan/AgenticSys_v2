Two chokepoints · one FirewallStack Every LLM call and every inter-agent edge passes through the stack
wrap(model).ainvoke(...)
LLM-call boundary
Chokepoint A
FirewallStack.wrap(ChatOpenAI(...)) returns a FirewalledModel. Every call site goes through await llm.ainvoke(system_prompt, user_message, tools, output_type).
Pre-step: case_scrubber.scrub replaces the literal active case-ID with <case> at word boundaries.
model.bind_tools(tools) if tools are passed — native OpenAI-style tool_calls.
Drives _tool_loop up to max_tool_turns = 12, dispatching tool_calls to a Python tool_map and appending ToolMessage back.
On FirewallRejection: log, increment attempt, append FIREWALL_GUIDANCE to the system prompt, re-sanitize and retry. Bounded by firewall.max_retries (default 2).
Returns LLMResult(status="success" | "blocked", data | error) — never raises to caller.
Each successful call appends a StepRecord(prompt, message, result, attempt) to firewall.step_history.
send(message, from, to)
Inter-agent transit boundary
Chokepoint B
await firewall.send(payload, from_agent, to_agent) is invoked on every cross-agent edge so redaction patterns apply even when content never hits the LLM — e.g., specialist outputs handed to the orchestrator.
Logs the transit (firewall_send event) with from, to, payload type.
_redact_payload walks the value: strings → _sanitize_message; Pydantic models round-trip via model_dump → model_validate; dicts/lists/tuples recurse; other types pass through.
Sanitization rules are exactly the same as the LLM boundary: CASE-\d+ → [CASE-ID] and \d{6,} → ***MASKED***.
Returns the redacted payload so callers can thread it into the next agent's input.

Seven mechanisms in v2 What · when · why it matters
Mechanism When Why
case_scrubber.scrub
replaces the active literal case-ID with <case>		 pre-invoke Production case IDs are bare digit strings; once mixed with other values they're indistinguishable from data. Scrubbing at the boundary keeps the LLM from ever seeing the raw ID, regardless of where it leaks in from.
_CASE_ID_RE
CASE-\d+ → [CASE-ID]		 both edges Normalizes the canonical "CASE-NNNN" form everywhere — applied by both _sanitize_message (LLM boundary) and send (inter-agent transit).
_DIGIT_RUN_RE
\d{6,} → ***MASKED***		 both edges Catches account numbers, long IDs, anything 6+ digits that could trip a content-safety filter or leak PII through a downstream redact step.
FIREWALL_GUIDANCE on retry
appended to system prompt, then re-prompt		 on rejection Tells the LLM why the previous response was blocked (raw account #, PII, role-injection like [SYSTEM], exec keywords) and asks for masked / descriptive language. Cheaper than expanding the regex set.
model.bind_tools(...)
native LangChain tool binding		 tool loop Uses OpenAI's structured tool_calls field — no prompt injection, no JSON-shape parsing in the adapter. _tool_loop dispatches each call to tool_map[name] and appends a ToolMessage(tool_call_id=...) back.
asyncio.Semaphore(concurrency_cap)
global cap across all wrapped models		 every invoke Default 8. Parallel fan-out (Reports + Team in parallel · per-pillar specialists in parallel) would otherwise trip provider rate limits — the semaphore is shared by every FirewalledModel under one FirewallStack.
firewall.send(msg, from, to)
redacts & logs every cross-agent edge		 inter-agent Specialist outputs, team drafts, review reports all flow through here on the way to the orchestrator. Pydantic models round-trip cleanly so type info survives the redaction pass.

FirewalledModel._tool_loop
LangChain bind_tools · OpenAI-style tool_calls · ToolMessage
When tools are passed to ainvoke, the wrapped model is bound via model.bind_tools(tools). The provider returns tool_calls as a typed field on the AIMessage; the loop dispatches each call to the matching Python function in tool_map, then appends a ToolMessage(content, tool_call_id) back to the messages list. Loop terminates when the response has no tool_calls, or after max_tool_turns = 12.
# inside FirewalledModel._tool_loop async with self.firewall._semaphore: response = await bound_model.ainvoke(messages) tool_calls = getattr(response, "tool_calls", None) if not tool_calls: return response # final AIMessage messages.append(response) # the AIMessage carrying tool_calls for tc in tool_calls: name = tc.get("name") if isinstance(tc, dict) else tc.name args = tc.get("args") if isinstance(tc, dict) else tc.args tc_id = tc.get("id") if isinstance(tc, dict) else tc.id fn = tool_map.get(name) result = str(fn(**(args or {}))) if fn else f"error: unknown tool {name}" messages.append(ToolMessage(content=result, tool_call_id=tc_id))
Execution lifecycle · where each call runs · LLM status at each beat All tool code runs in Python · FirewallRejection bubbles up from upstream provider/policy
One semaphore, many models
firewall._semaphore is shared across every FirewalledModel built from the same FirewallStack. Parallel specialists + parallel report scan all draw from the same 8 slots.
FirewallRejection is upstream-typed
The wrapper doesn't raise blocks — it raises only inside the inner provider integration; the wrapper catches and translates to LLMResult(status="blocked") after retries are exhausted.
Two retry layers · independent budgets
LangChain's ChatOpenAI(max_retries=2) handles 5xx/429/timeout. firewall.max_retries=2 handles content-safety blocks. They compose multiplicatively in the worst case.

Beat LLM status Python action · code execution locus messages size
T1 idle case_scrubber.scrub(user_message, case_id) — boundary scrub of the active case-ID literal. —
T2 idle bound_model = model.bind_tools(tools) + build [SystemMessage, HumanMessage]. 2
T3 inferring · in semaphore async with firewall._semaphore: first bound_model.ainvoke(messages) — provider returns AIMessage, possibly with tool_calls. 2
T4 idle · Python runs tool Code executes in Python: for each tool_call, tool_map[name](**args) — e.g. query_table('model_scores', ...) hits the in-process DataGateway. 2 → 3
T5 idle Append ToolMessage(content=result, tool_call_id=tc_id) for each call. AIMessage from T3 is also appended before the ToolMessages. 2 + 1 + N
T6 inferring · larger context Loop iteration — re-acquire semaphore, re-invoke bound_model. May emit more tool_calls or a final response. prev + 1
… idle ↔ inferring Loop repeats up to max_tool_turns = 12. Each round adds 1 AIMessage + N ToolMessages. +1+N / round
Tn idle · terminal Final AIMessage (no tool_calls). Adapter parses content via output_type(**parsed) when set, else returns {response: content}. Appends StepRecord. final
Tr blocked · retry If FirewallRejection raised inside _tool_loop: log, increment attempt, append FIREWALL_GUIDANCE to current_system, re-sanitize current_message, restart the outer while loop with fresh [SystemMessage, HumanMessage]. reset to 2
