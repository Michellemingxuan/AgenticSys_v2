Shared Specialist Layer — same agent class, YAML-differentiated behavior per pillar
Shared Specialist Pool
Each specialist is one agent class with three prompt-mode skills.
Which specialists run, and how they are focused, is controlled by each pillar's YAML.
How YAML controls the same specialist across pillars:
The Bureau specialist runs identically in Credit&Risk and CBO — but its focus, table_scope, and prompt_overlay differ. Same class, different config = different analytical emphasis.

Data Agent Pool — general-purpose, stateless
Each data agent is identical: it receives a validated, redacted data request from the repo agent and executes it.
The repo agent routes across the pool. Multiple agents run in parallel for concurrent specialist requests.
Pillar-awareness lives in the YAML config, not in the data agents themselves.

YAML is the single control plane. A pillar config declares which specialists to call, their analytical focus, table scope, and prompt overlays. Adding a new pillar = one new YAML file. Changing how Bureau analyses a case for CBO vs Credit & Risk = edit a YAML field. No Python changes required. The specialist agent class, orchestrator agent class, and data agents are all reusable as-is.
