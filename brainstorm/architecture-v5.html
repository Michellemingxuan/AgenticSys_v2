Shared Skills
All specialist agents expose the same three prompt-mode skills. Behavior is differentiated by pillar instructions injected at call time.
Report
Generate a full section narrative. Output: Key Findings → Evidence → Risk Implication.
Answer
Answer a targeted sub-question from the orchestrator. Also used in peer review mode to flag contradictions in a peer's output.
Data Request
Declaratively specify required data — tables, columns, and filters. Routed to the Repository Agent for validation, gating, and redaction.

Same Python Class. YAML controls analytical focus, table scope, prompt overlay, and blocked fields. Adding a new pillar or retuning a specialist = one YAML change, no code changes.

Shared Skills — Data Agents
All data agents expose the same four skills.
Code Gen
Generate executable Python / SQL from the validated declarative data request.
Execute
Run generated code in a sandboxed environment. Retry on recoverable failures.
Retrieve
Return structured results to the requesting specialist via the Repository Agent.
Analyze
Optional: produce a brief statistical summary or anomaly flag on retrieved data before returning.
How Specialist Selection Works. YAML provides pillar context — analytical focus, table scope, prompt overlays, and blocked data fields. The orchestrator's Team Construction skill reads both the pillar instructions and the reviewer's question, applies compliance filters, then dynamically selects which specialists are relevant for each sub-question. There is no hard-coded specialist list. Pillar instructions are injected directly into each called specialist's prompt at runtime.
